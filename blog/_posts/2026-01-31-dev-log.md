        # [dev-log-hub] Qiita 자동 발행 시스템 구축: n8n과 qiita-cli 활용기

안녕하세요! `dev-log-hub` 프로젝트에서 중요한 자동화 기능을 성공적으로 도입하여 그 경험을 공유하고자 합니다. 이번 업데이트를 통해 반복적이고 수동적이었던 Qiita 글 발행 과정을 자동화하여 개발 효율성을 크게 향상시켰습니다.

## 🚀 왜 자동화가 필요했을까요?

`dev-log-hub`는 개발 일지를 효율적으로 관리하고 공유하는 것을 목표로 합니다. 하지만, 작성된 개발 일지를 Qiita에 수동으로 발행하는 과정은 다음과 같은 문제점을 안고 있었습니다.

*   **시간 소모:** 매번 로그인하여 마크다운을 복사-붙여넣기하고, 태그를 설정하는 등의 작업이 반복적으로 이루어졌습니다.
*   **휴먼 에러 가능성:** 수동 작업은 실수할 여지가 항상 존재합니다. (예: 잘못된 태그, 오타 등)
*   **일관성 부족:** 발행 주기가 일정하지 않거나, 특정 담당자가 부재 시 발행이 지연될 수 있었습니다.

이러한 문제들을 해결하고 더 나은 개발 경험을 제공하기 위해 자동화 시스템 구축을 결정했습니다.

## ✨ n8n과 qiita-cli를 활용한 자동화 워크플로우

이번 자동화 시스템의 핵심은 `n8n`과 `qiita-cli` 두 가지 도구입니다.

### 1. `qiita-cli`: Qiita 게시글 발행을 위한 CLI 도구

`qiita-cli`는 명령줄 인터페이스를 통해 Qiita에 게시글을 쉽게 발행할 수 있도록 돕는 공식 도구입니다. 로컬에 저장된 마크다운 파일을 직접 업로드하거나, 특정 옵션을 지정하여 게시글을 관리할 수 있습니다.

```bash
# qiita-cli 설치 예시 (설치되어 있다고 가정)
npm install -g qiita-cli

# 로컬 마크다운 파일 발행 (예시)
qiita new --title "오늘의 개발일지" --tags "개발, 자동화" --file "./docs/2023-10-27-devlog.md" --private false
```

### 2. `n8n`: 유연한 워크플로우 자동화 도구

`n8n`은 다양한 웹 서비스와 애플리케이션을 연결하여 복잡한 워크플로우를 시각적으로 구축하고 자동화할 수 있는 강력한 도구입니다. 특정 이벤트 트리거(예: 새로운 파일 생성, 스케줄링)에 따라 일련의 작업을 수행하도록 설정할 수 있습니다.

`dev-log-hub`에서는 `n8n`을 활용하여 다음과 같은 워크플로우를 구현했습니다.

1.  **트리거:** 특정 조건(예: Git 저장소에 새 개발 일지 마크다운 파일 푸시, 스케줄링)이 충족될 때 워크플로우를 시작합니다.
2.  **데이터 가공:** 발행할 개발 일지 내용을 불러와 `qiita-cli`가 요구하는 형식에 맞게 가공합니다. (예: 제목, 태그 추출)
3.  **`qiita-cli` 실행:** `n8n`의 'Execute Command' 또는 'HTTP Request' 노드를 활용하여 `qiita-cli` 명령을 실행하고, 준비된 개발 일지를 Qiita에 발행합니다.
4.  **피드백 및 로깅:** 발행 성공 여부를 확인하고, 필요한 경우 슬랙(Slack) 등의 메신저로 알림을 보내거나 로그를 기록합니다.

*(n8n의 내부 워크플로우 정의는 코드 diff에 포함되지 않아, 개념적인 설명으로 대체합니다.)*

## ✅ 수많은 테스트를 통한 안정성 확보

이번 자동화 기능은 `feat: first automated publish via qiita-cli and n8n` 커밋을 통해 처음 도입되었습니다. 이후 `test: trigger social publish` 커밋이 여러 차례 반복된 것을 보실 수 있습니다. 이는 처음 구축된 자동화 워크플로우가 안정적으로 동작하는지, 예외 상황에서도 잘 처리되는지 등을 면밀히 테스트하고 검증하는 과정이었습니다. 덕분에 믿고 사용할 수 있는 자동화 시스템을 구축할 수 있었습니다.

## 💡 결론 및 향후 계획

`dev-log-hub` 프로젝트는 이번 `qiita-cli`와 `n8n`을 활용한 자동 발행 시스템 구축을 통해 개발 일지 발행의 효율성과 편의성을 크게 향상시켰습니다. 이제 개발자들은 컨텐츠 작성에만 집중하고, 발행에 대한 부담을 덜 수 있게 되었습니다.
